# Todo AI Workflow

## Когда использовать
- Пользователь просит "взять задачу", "выполни задачу из доски", "поработай с kanban"
- Пользователь говорит "что у нас в работе", "какие задачи"
- Нужно взаимодействовать с todo-ai доской

## Проверка конфига (Шаг 0)

Перед началом работы убедись что конфиг проекта корректен:
- Проверь файл `.qwen/todo-ai.json` в корне проекта
- Если файла нет — создай: `{"board": "main"}`
- Если файл есть но невалиден — исправь
- Используй команду `/todo-config`

## MCP Инструменты

| Инструмент | Описание |
|---|---|
| `todo_next_run` | Взять задачу из "Сделать" → "В работе" |
| `todo_next` | Посмотреть следующую задачу (без изменения статуса) |
| `todo_list` | Список задач из "Бэклог-AI" |
| `list_tasks` | Список задач с фильтром по статусу |
| `get_task(task_id)` | Получить задачу по ID |
| `update_task(task_id, ...)` | Обновить поля задачи |
| `write_to_notepad(task_id, note)` | Записать результаты выполнения |
| `append_log(task_id, log_entry)` | Добавить запись в лог |
| `change_status(task_id, status)` | Переместить задачу (5=Готово, 4=Требует внимания) |
| `create_task(title, ...)` | Создать задачу в Бэклог-AI |
| `delete_task(task_id)` | Удалить задачу |

## Пошаговый процесс выполнения задачи

### Шаг 1: Взять задачу
```
todo_next_run() → получаем задачу из "Сделать" → "В работе"
```

### Шаг 2: Изучить задачу
```
get_task(task_id) → читаем title, description, context_ai, checklist, ai_notepad
```

### Шаг 3: Выполнить задачу
- Если задача понятна и выполнима — выполняй
- Если нужен дополнительный контекст — запиши в `write_to_notepad` что нужно
- Если задача требует подзадач — используй `update_task` с чеклистом
- Если задача слишком сложная — перемести в "Требует внимания" (status=4)

### Шаг 4: Записать результаты
```
write_to_notepad(task_id, "Результат выполнения: ...")
append_log(task_id, "Задача выполнена AI-агентом. Результаты: ...")
```

### Шаг 5: Закрыть задачу
```
change_status(task_id, 5)  # 5 = Готово
```

Или если проблемы:
```
change_status(task_id, 4)  # 4 = Требует внимания
```

## Статусы доски

| Код | Название | Кто может |
|---|---|---|
| 0 | Бэклог | ТОЛЬКО human |
| 1 | Бэклог-AI | ТОЛЬКО ai |
| 2 | Сделать | Любой |
| 3 | В работе | Любой |
| 4 | Требует внимания | Любой |
| 5 | Готово | Любой |
| 6 | Архив | Любой |

## Правила
1. ВСЕГДА записывай результаты в notepad перед закрытием
2. ВСЕГДА логируй действие через append_log
3. НЕ создавай задачи в status=0 (только human)
4. НЕ создавай задачи в status=1 напрямую (используй create_task)
5. Если задача невыполнима — перемести в status=4 с пояснением

## Пример полного цикла
1. `todo_next_run()` → задача #5 "Написать функцию X"
2. `get_task(5)` → читаем контекст
3. Выполняем задачу (пишем код, тесты, документацию)
4. `write_to_notepad(5, "Функция X реализована в src/utils.py. Протестирована.")`
5. `append_log(5, "AI-агент выполнил задачу. Функция готова.")`
6. `change_status(5, 5)` → задача закрыта
