MCP инструменты

11 инструментов для AI-агентов

Назад к Kanban-доске

Endpoint

POST http://localhost:8167/api/mcp/tools/call

Body: {"name": "tool_name", "arguments": {...}}

Задачи

create_task

Создать задачу в Бэклог-AI (status=1). Чеклист передаётся как JSON-объект.

{"title": "...", "description": "...", "context_ai": "...", "checklist": {"title": "План", "tasks": [{"title": "Шаг 1", "done": false, "children": []}]}}

update_task

Обновить поля задачи, включая чеклист.

{"task_id": 1, "title": "...", "checklist": {"title": "Новый", "tasks": []}}

get_task

Получить задачу по ID. Ответ включает полный чеклист с вложенностью.

{"task_id": 1}

list_tasks

Список задач с фильтром по статусу (0-6).

{"status": 2}

delete_task

Удалить задачу.

{"task_id": 1}

write_to_notepad

Добавить заметку в AI-блок задачи.

{"task_id": 1, "note": "Прогресс: ..."}

append_log

Добавить запись в лог задачи.

{"task_id": 1, "log_entry": "Выполнено: ..."}

change_status

Изменить статус задачи (переместить между колонками).

{"task_id": 1, "status": 5}

Управление

todo_next

Показать следующую задачу из "Сделать" (status=2). Только просмотр, без изменений.

{}

todo_next_run

Взять задачу из "Сделать" и переместить в "В работе" (status 2 -> 3). Возвращает полный объект задачи с чеклистом.

{}

todo_list

Показать задачи из "Бэклог-AI" (status=1). До 50 штук с кратким описанием.

{"limit": 50}

Чеклист в задаче

Чеклист хранится как JSON-объект внутри задачи. Доступ через create_task/update_task/get_task. Отдельных MCP инструментов для чеклиста нет.

{
  "checklist": {
    "title": "План работ",
    "tasks": [
      {
        "title": "Настроить БД",
        "done": false,
        "children": [
          {"title": "Создать таблицы", "done": true, "children": []}
        ]
      }
    ]
  }
}

WebSocket

ws://localhost:8167/ws

{"type": "task_change", "action": "status_changed", "task_id": 1, "task": {...}}
