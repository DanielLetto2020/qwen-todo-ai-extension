# /todo-next-run — Полный цикл задачи

Выполни полный цикл задачи от начала до завершения:

## Шаг 1 — Взять задачу в работу
Используй MCP инструмент `todo_next_run` (без параметров). Он автоматически:
- Находит задачу в колонке "Сделать" (status=2) которая находится в самом верху списка (наивысший приоритет, наименьший id_task)
- Перемещает её в колонку "В работе" (status=3)
- Возвращает объект задачи со всеми полями

## Шаг 2 — Выполнить задачу
- Прочитай все поля задачи из результата `todo_next_run`:
  - `title` — заголовок задачи
  - `description` — подробное описание
  - `context_ai` — дополнительный контекст для AI
  - `checklist` — чеклист в формате `{"title": "...", "tasks": [{"title": "...", "done": false}]}`
  - `ai_notepad` — заметки от предыдущих выполнений
- Выполни то что описано в задаче
- Если есть чеклист — пройди по всем пунктам, отмечай выполненные через MCP `update_task` обновляя поле `checklist`

## Шаг 3 — Завершить задачу
### Если задача выполнена успешно:
Используй MCP `change_status` с параметрами:
- `task_id` — ID задачи (id_task из шага 1)
- `status: 5` — перемещает в колонку "Готово"

### Если не получилось выполнить:
Используй MCP `change_status` с параметрами:
- `task_id` — ID задачи
- `status: 4` — перемещает в колонку "Требует внимания"

После смены статуса подробно объясни что пошло не так и почему задача требует внимания.

**ВАЖНО:**
- НЕ задавай вопросов типа "Хочешь я возьму в работу?" или "Хочешь выполнить?" — команда уже означает действие
- НЕ останавливайся после взятия задачи — выполни её до конца
- ВСЕГДА записывай результаты выполнения в `ai_notepad` через MCP `write_to_notepad`:
  - `task_id` — ID задачи
  - `note` — краткое описание что было сделано, дата, результаты
- ВСЕГДА логируй ключевые действия через MCP `append_log`:
  - `task_id` — ID задачи
  - `log_entry` — описание действия (например: "Начато выполнение", "Завершено: ...")
- Если задача требует контекста которого нет — уточни у пользователя

## Статусы колонок (для справки):
| Код | Колонка |
|-----|---------|
| 2 | Сделать |
| 3 | В работе |
| 4 | Требует внимания |
| 5 | Готово |

## Используемые MCP инструменты:
- `todo_next_run` — взять задачу из "Сделать" → "В работе"
- `change_status(task_id, status)` — сменить статус задачи
- `write_to_notepad(task_id, note)` — записать результаты
- `append_log(task_id, log_entry)` — записать в лог
- `update_task(task_id, ...)` — обновить поля задачи (например checklist)

После выполнения покажи пользователю:
- Номер задачи (#id_task)
- Название
- Что было сделано (кратко, по существу)
- Финальный статус (Готово или Требует внимания)
