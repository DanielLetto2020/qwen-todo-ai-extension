# /todo-start-ui — Запустить UI и документацию

Проверь запущен ли HTTP сервер todo-ai на порту 8167 и открой документацию.

## Шаг 1 — Проверить запущен ли сервер
Выполни команду:
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:8167/health
```
Если ответ `200` — сервер уже запущен, переходи к Шагу 3.
Если ответ не `200` или ошибка — переходи к Шагу 2.

## Шаг 2 — Запустить сервер
Запусти сервер в фоновом режиме:
```bash
cd ~/.qwen/extensions/todo-ai && EXTENSION_PATH=~/.qwen/extensions/todo-ai python -m server.main &
```
Подожди 3 секунды и проверь что сервер запустился:
```bash
curl -s http://localhost:8167/health
```
Если получил ответ `{"status":"ok",...}` — сервер работает.

## Шаг 3 — Открыть документацию
Открой следующие URL через MCP Playwright (используй `open_new_tab` для каждого):

1. **Главная UI** — http://localhost:8167/
   - Это Kanban-доска с задачами

2. **Документация приложения** — http://localhost:8167/app-docs
   - Полная документация по UI

3. **MCP инструменты** — http://localhost:8167/mcp-tools
   - Справка по MCP API

4. **Настройки** — http://localhost:8167/settings
   - Настройки приложения

5. **Локальный README** — файл `~/.qwen/extensions/todo-ai/README.md`
   - Прочитай содержимое файла и покажи пользователю краткое описание

После открытия всех вкладок сообщи пользователю:
- ✅ Сервер запущен на http://localhost:8167
- ✅ Документация открыта в браузере
- 📖 Краткое описание проекта из README.md

**ВАЖНО:**
- Если сервер уже запущен — НЕ перезапускай его
- Если порт 8167 занят другим процессом — сообщи об этом пользователю
- Для проверки порта используй: `ss -tlnp | grep 8167`
- Документацию открывай в отдельных вкладках браузера
