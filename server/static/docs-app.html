1 О проекте

Что такое Todo AI?

Todo AI — это расширение для Qwen Code CLI, которое добавляет полноценную Kanban-доску для управления задачами. Оно работает в двух режимах:

  • ▸ MCP stdio — AI-агент управляет задачами прямо из чата Qwen Code, используя 11 специальных инструментов
  • ▸ Веб-UI — красивая Kanban-доска в браузере с drag-and-drop, заметками и визуальным управлением

Для чего это нужно?

Если вы используете Qwen Code для разработки, Todo AI помогает:

  • ▸ Разбивать большие задачи на маленькие подзадачи
  • ▸ Видеть прогресс работы визуально (Kanban-доска)
  • ▸ Делегировать задачи AI-агенту — он сам берёт, выполняет и отчитывается
  • ▸ Вести историю всех изменений (кто, когда, что сделал)
  • ▸ Управлять несколькими проектами через разные доски
Kanban-доска

7 колонок с drag-and-drop, сортировкой и фильтрацией

MCP API

2 Архитектура

Todo AI состоит из нескольких компонентов:

MCP stdio

Qwen Code вызывает инструменты через stdin/stdout (JSON-RPC). Это основной режим — AI управляет задачами прямо из чата.

FastAPI HTTP

Веб-сервер с UI в браузере + REST API. Запускается отдельно для визуального управления доской.

SQLite

Лёгкая база данных в файле data/tasks.db. Все задачи хранятся здесь, изолированы по доскам (board).

WebSocket

Realtime обновления в UI. Когда AI меняет задачу через MCP, UI обновляется мгновенно без перезагрузки страницы.

3 Колонки доски (статусы)

Каждая задача на доске находится в одной из 7 колонок. Перемещение между колонками = изменение статуса.

0 Бэклог человек
1 Бэклог-AI AI
2 Сделать любой
3 В работе любой
4 Требует внимания любой
5 Готово любой
6 Архив любой

Важно: Статус 0 (Бэклог) могут создавать только люди. Статус 1 (Бэклог-AI) — только AI-агенты. Это предотвращает хаос на доске.

4 Установка расширения

Требования

  • ▸ Qwen Code CLI — установлен и настроен
  • ▸ Python 3.10+ — для запуска MCP сервера
  • ▸ Git — если установка из репозитория

Способ 1: Из Git-репозитория (рекомендуется)

Самый простой способ. Клонируйте репозиторий расширения:

qwen extensions install https://github.com/your-username/todo-ai

Или если репозиторий на GitHub:

qwen extensions install your-username/todo-ai

Способ 2: Из локальной папки

Если у вас есть исходники расширения:

qwen extensions install /путь/к/todo-ai

Способ 3: Для разработки (symlink)

Создаёт символическую ссылку — изменения применяются сразу без update:

qwen extensions link /путь/к/todo-ai

Установка зависимостей Python

Перейдите в папку расширения и установите зависимости:

cd ~/.qwen/extensions/todo-ai pip install -r server/requirements.txt

После установки перезапустите сессию Qwen Code, чтобы расширение активировалось. Все изменения в расширениях вступают в силу только после перезапуска.

5 Настройка доски

Настройка для конкретного проекта

Каждый проект может иметь свою доску. Создайте файл .qwen/todo-ai.json в корне вашего проекта:

{
  "board": "my-project-name"
}

Теперь все задачи будут привязаны к доске my-project-name. По умолчанию используется доска main.

Проверка установки

Убедитесь что расширение загружено — введите в Qwen Code:

/extensions

Вы должны увидеть todo-ai в списке активных расширений.

6 Обновление и удаление

Обновление расширения

qwen extensions update todo-ai

Или обновить все расширения сразу:

qwen extensions update --all

Удаление расширения

qwen extensions uninstall todo-ai

Отключение / Включение

Можно временно отключить расширение без удаления:

qwen extensions disable todo-ai qwen extensions enable todo-ai

7 Запуск и остановка

MCP stdio (основной режим)

Не требует ручного запуска! MCP сервер в режиме stdio запускается автоматически Qwen Code при каждом обращении к инструменту. Это прописано в qwen-extension.json расширения. Просто используйте команды /todo, /todo-next и т.д.

HTTP UI (веб-интерфейс)

Для запуска визуальной Kanban-доски в браузере:

cd ~/.qwen/extensions/todo-ai python -m server.main # порт 8167 python -m server.main --port 9000 # свой порт

Или через env: TODO_AI_APP_PORT=9000 python -m server.main

Откройте в браузере: http://localhost:8167

Остановка UI сервера

Нажмите Ctrl+C в терминале где запущен сервер. Или найдите и убейте процесс:

# Найти процесс ps aux | grep "server.main" | grep -v grep # Убить процесс (замените PID на реальный) kill PID

Совет: Запустите UI сервер в фоновом режиме (& в конце команды или nohup), чтобы он работал постоянно.

8 Мультидоски

Каждый проект может иметь свою отдельную Kanban-доску с независимыми задачами.

Создать новую доску

В UI: кнопка ＋ рядом с селектором досок в верхнем левом углу.

Через API:

curl -X POST http://localhost:8167/api/boards -H "Content-Type: application/json" -d '{"name":"my-project"}'

Переключить доску

В UI: выпадающий список в верхнем левом углу. Или через API:

curl -X PUT http://localhost:8167/api/boards/active -H "Content-Type: application/json" -d '{"board":"my-project"}'

Список всех досок

curl http://localhost:8167/api/boards

Удалить доску

curl -X DELETE http://localhost:8167/api/boards/my-project

Доска main не может быть удалена — это доска по умолчанию.

10 Команды Qwen Code (слэш-команды)

Введите эти команды прямо в чате Qwen Code:

Команда Что делает Пример
/todo Создать новую задачу в Бэклог-AI. AI спросит заголовок и описание. /todo → "Рефакторинг API"
/todo-list Показать список задач из Бэклог-AI (по умолчанию) или с фильтром по статусу. /todo-list --status 2
/todo-next Показать следующую задачу из колонки "Сделать" (только просмотр, статус не меняется). /todo-next
/todo-next-run Взять следующую задачу из "Сделать" и переместить в "В работе" (статус 2 → 3). /todo-next-run

Подсказка: Команды расширения имеют наименьший приоритет. Если у вас есть своя команда /todo, используйте /todo-ai.todo для вызова команды из этого расширения.

11 MCP инструменты для AI-агентов

AI-агент автоматически использует эти инструменты через MCP протокол. Вам не нужно вызывать их вручную — AI делает это сам.

Управление задачами

create_task status=1

Создать задачу в колонке "Бэклог-AI". Только для AI-агентов.

Параметры: title (обязательно), description, context_ai, checklist

update_task

Обновить поля существующей задачи (заголовок, описание, статус и т.д.).

Параметры: task_id (обязательно), title, description, status, context_ai

get_task

Получить полную информацию о задаче по её ID.

Параметры: task_id

list_tasks

Получить список задач с фильтром по статусу.

Параметры: status (опционально, 0-6)

delete_task

Удалить задачу по ID.

Параметры: task_id

change_status

Изменить статус задачи (переместить в другую колонку).

Параметры: task_id, status (0-6)

Заметки и логирование

write_to_notepad

Записать заметку в AI-блок задачи. Результаты выполнения пишутся сюда.

Параметры: task_id, note

append_log

Добавить запись в лог истории задачи. Каждое действие логируется.

Параметры: task_id, log_entry

Управление рабочим процессом

todo_next

Показать следующую задачу из "Сделать". Только просмотр, статус не меняется.

todo_next_run

Взять следующую задачу из "Сделать" → "В работе" (status 2 → 3).

todo_list

Показать задачи из "Бэклог-AI". До 50 задач с кратким описанием.

12 Правила для AI-агента

✕

НЕ создавай задачи в статусе 0 (Бэклог) — это колонка только для людей.

✓

Создавай задачи в статусе 1 (Бэклог-AI) через create_task.

✓

ВСЕГДА записывай результаты выполнения в ai_notepad через write_to_notepad.

✓

ВСЕГДА логируй действия через append_log — так ведётся полная история.

⚠

Если задача невыполнима — перемести в статус 4 (Требует внимания) и объясни причину в ai_notepad.

13 Скилы (Skills)

Скилы — это наборы экспертных знаний, которые AI-агент использует автоматически. В отличие от команд, скилы активируются самим AI когда он считает это нужным.

Как это работает

  • ▸ AI анализирует ваш запрос и описание скила
  • ▸ Если запрос совпадает с описанием — AI автоматически применяет скил
  • ▸ Можно вызвать явно: /skills <имя-скила>
  • ▸ Список доступных скилов: /skills

Расположение: Скилы расширения находятся в папке skills/ внутри расширения. Каждый скил — это папка с файлом SKILL.md.

14 Полный цикл работы с задачей

Вот как выглядит типичный рабочий процесс — от создания задачи до завершения:

1

Создание задачи

Пользователь или AI создаёт задачу через команду /todo или инструмент create_task.

/todo → "Рефакторинг модуля авторизации"

Задача появляется в колонке Бэклог-AI (status=1).

2

Перенос в «Сделать»

Когда задача готова к выполнению — перенесите её в колонку "Сделать" (status=2).

В UI: drag-and-drop или стрелка → на карточке. Через MCP: change_status(task_id=1, status=2)

3

AI берёт задачу в работу

Команда /todo-next-run или инструмент todo_next_run берёт следующую задачу из "Сделать" и переносит в "В работе" (status 2 → 3).

/todo-next-run
4

AI выполняет задачу

AI-агент читает описание задачи, пишет код, тестирует. В процессе он:

  • ▸ Записывает промежуточные результаты в ai_notepad
  • ▸ Логирует действия через append_log
5

Задача завершена

AI перемещает задачу в "Готово" (status=5) и записывает итоговый результат.

change_status(task_id=1, status=5)

15 Примеры использования

Пример 1: Разбивка большой задачи на подзадачи

Вы хотите чтобы AI рефакторил модуль авторизации. Создайте задачи через MCP:

AI создаёт через create_task:
  - "Рефакторинг: модель User"
  - "Рефакторинг: контроллер AuthController"
  - "Рефакторинг: middleware авторизации"
  - "Написание тестов для AuthController"

Все задачи попадают в Бэклог-AI (status=1)
Потом /todo-next-run берёт первую задачу в работу

Пример 2: AI отчитывается о выполнении

После выполнения задачи AI записывает результат:

# AI записывает результат
write_to_notepad(task_id=5, note="✅ Готово: добавлена валидация email и пароля")
append_log(task_id=5, log_entry="Реализовано: bcrypt хеширование паролей, валидация email по RFC 5322")

# Переносит в "Готово"
change_status(task_id=5, status=5)

Пример 3: Проблема — задача перенесена в «Требует внимания»

Если AI не может выполнить задачу, он сигнализирует:

write_to_notepad(task_id=8, note="⚠ Не могу выполнить: нет доступа к API платёжной системы. Нужен API ключ.")
append_log(task_id=8, log_entry="Задача заблокирована: требуется API ключ от платёжной системы")
change_status(task_id=8, status=4)  # Требует внимания

Пример 4: Работа с несколькими проектами

У вас два проекта — backend и frontend:

# В папке backend/.qwen/todo-ai.json:
{"board": "backend"}

# В папке frontend/.qwen/todo-ai.json:
{"board": "frontend"}

16 REST API справочник

Если HTTP сервер запущен, доступны следующие endpoints:

Задачи — /api/tasks

GET /api/tasks — список задач (?status=N)
GET /api/tasks/{id} — задача по ID
POST /api/tasks — создать задачу
PUT /api/tasks/{id} — обновить задачу
PATCH /api/tasks/{id}/status — изменить статус
DEL /api/tasks/{id} — удалить задачу

Доски — /api/boards

GET /api/boards — список досок
GET /api/boards/active — текущая доска
PUT /api/boards/active — переключить доску
POST /api/boards — создать доску
DEL /api/boards/{name} — удалить доску

Заметки — /api/notes

GET /api/notes — список заметок
POST /api/notes — создать заметку
PUT /api/notes/{id} — обновить заметку
DEL /api/notes/{id} — удалить заметку

MCP — /api/mcp

GET /api/mcp/tools — список инструментов
GET /api/mcp/docs — документация MCP
POST /api/mcp/tools/call — вызвать инструмент

Другие endpoints

GET /api/board — колонки + задачи текущей доски
GET /health — проверка здоровья сервера
GET /config — конфигурация приложения
GET /api/db/tables — список таблиц БД
GET /api/db/table/{name} — содержимое таблицы

17 Структура задачи 

{
  "id": 139,           // Внутренний ID базы данных (autoincrement)
  "id_task": 42,       // Бизнес-ID задачи (уникальный в рамках доски) ← ИСПОЛЬЗУЙТЕ ЭТОТ ID
  "title": "Название задачи",
  "description": "Подробное описание...",
  "context_ai": "Контекст для AI-агента (инструкции, ограничения)",
  "status": 1,         // 0-6 (см. колонки доски)
  "type": "base",      // Тип задачи
  "priority": 0,       // Приоритет сортировки (меньше = выше)
  "log": "[...]",      // JSON-массив с историей изменений
  "ai_notepad": "...", // Заметки AI (результаты выполнения)
  "checklist": {       // Вложенный чеклист
    "title": "Чеклист",
    "tasks": [{"id": "...", "title": "Пункт", "done": false, "children": []}]
  },
  "board": "main",     // Имя доски
  "created_at": "...", // Дата создания
  "updated_at": "..."  // Дата последнего обновления
}

Важно: Все MCP инструменты работают с id_task (бизнес-ID), а не с внутренним id. При создании задачи create_task возвращает task_id — это и есть id_task.

18 Устранение проблем

Расширение не загружается

▸ Проверьте что расширение в списке: /extensions

▸ Убедитесь что Python установлен: python --version

▸ Установите зависимости: pip install -r server/requirements.txt

▸ Перезапустите Qwen Code

MCP инструменты не работают

▸ Проверьте что qwen-extension.json настроен правильно

▸ Убедитесь что переменные окружения заданы (DATABASE_URL, EXTENSION_PATH)

▸ Очистите кэш Python: find . -name "__pycache__" -exec rm -rf {} +

▸ Все инструменты работают по id_task, не по id

UI не открывается

▸ Запустите сервер: python -m server.main --port 8167

▸ Проверьте что порт свободен: curl http://localhost:8167/health

▸ Убедитесь что зависимости установлены: fastapi, uvicorn, pydantic

Задача не находится по ID

▸ Используйте id_task (бизнес-ID), а не id базы данных

▸ id_task показывается в UI на каждой карточке (например #42)

▸ Проверьте что задача принадлежит текущей доске (board)
