# Todo AI — Kanban-доска для AI-агентов

## Обзор
Todo AI — это расширение Qwen Code которое предоставляет Kanban-доску с MCP инструментами для управления задачами. Поддерживает **мультиборды** — каждая доска изолирована по `board`.

## Как это работает

### Архитектура

Два независимых процесса работают с одной БД:

- **UI (браузер)** — показывает ВСЕ доски, переключается между ними
- **AI-агент (MCP stdio)** — запускается автоматически Qwen Code, читает `.qwen/todo-ai.json` проекта → работает со своей доской
- **SQLite** — одна база, задачи изолированы по полю `board`

### Колонки доски (статусы)

| Код | Название | Цвет | Кто создаёт |

| 0 | Бэклог | #3B82F6 | human |

## Доступные команды

| Команда | Описание |
|---|---|
| `/todo` | Создать задачу в Бэклог-AI |
| `/todo-config` | Проверить/создать `.qwen/todo-ai.json` |
| `/todo-next` | Показать следующую задачу из "Сделать" |
| `/todo-next-run` | Взять задачу из "Сделать" → выполнить → "Готово" |
| `/todo-list` | Список задач (по умолчанию: Бэклог-AI) |
| `/todo-start-ui` | Запустить UI сервер и открыть документацию |

## MCP Инструменты (11 штук)

### Задачи
- `create_task(title, description?, context_ai?, checklist?)` — создать в Бэклог-AI (status=1)
- `update_task(task_id, ...)` — обновить поля
- `get_task(task_id)` — получить задачу
- `list_tasks(status?)` — список с фильтром
- `delete_task(task_id)` — удалить

### Заметки и лог
- `write_to_notepad(task_id, note)` — заметка AI
- `append_log(task_id, log_entry)` — запись в лог

### Управление
- `change_status(task_id, status)` — сменить статус
- `todo_next()` — посмотреть следующую из "Сделать"
- `todo_next_run()` — взять из "Сделать" → "В работе"
- `todo_list(limit?)` — задачи из "Бэклог-AI"

## Мультиборды
Каждый проект может иметь свою доску. Конфигурация в `.qwen/todo-ai.json`:
```json
{
  "board": "my-project-name"
}
```

AI-агент автоматически читает этот конфиг при запуске и работает только со своей доской.

> Доска `main` — системная, создаётся автоматически. Её нельзя удалить или переименовать.

## Правила для AI
1. **НЕ** создавай задачи в status=0 (Бэклог) — это только для людей
2. Создавай задачи в status=1 (Бэклог-AI) через `create_task`
3. ВСЕГДА записывай результаты в `ai_notepad` перед закрытием
4. ВСЕГДА логируй через `append_log`
5. Если задача невыполнима → status=4 (Требует внимания)

## Запуск UI
Из любого проекта используй команду:
```
/todo-start-ui    # Проверит, запустит сервер и откроет документацию
```

Вручную (если нужно):
```bash
python -m server.main              # порт 8167
python -m server.main --port 9000  # свой порт
```
Или через env: `TODO_AI_APP_PORT=9000 python -m server.main`

> Приоритет: `--port` > env `TODO_AI_APP_PORT` > `8167`

### Документация в браузере
После запуска UI доступны:
- `http://localhost:8167/` — Kanban-доска
- `http://localhost:8167/app-docs` — Документация приложения
- `http://localhost:8167/mcp-tools` — Справка по MCP API
- `http://localhost:8167/settings` — Настройки
